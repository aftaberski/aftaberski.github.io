All About That Class

NOVEMBER 30, 2014

So far we've just been able to call generic program methods. I know I've been secretly wondering why I can't just call those methods like eat cookie instead of cookie.eat. With the help of classes, we can actually create methods and call them like that, as long as the object is a member of the same class as the instance method. Basically, Classes let us organize our objects so we can apply certain characteristics and abilities to an entire group of objects. That's pretty vague, so let's explore that a little bit more.

A Brief Intro To Classes

First things first, Instance Variables are often used in classes. They always begin with an @ sign. That's how you can create them and spot them. These little guys are pretty special because they last as long as the object does (unlike a method's local variables which only last until a method is finished). For example:

class PhoneNumber

  def initialize number
    @number = number
  end

  def dial
    @number_array = @number.to_s.split("")
    @number_array.each{ |num| p num.to_i; sleep 0.3}
  end
end

phone_call = PhoneNumber.new(2125559566) #=>creates a new instance of the class
phone_call.dial
#=> 2
    1
    2
    5
    5
    5
    9
    5
    6
    6

Pretty cool, right? This means we can create a new instance of the PhoneNumber class for any number we would like to dial. .dial is an example of an Instance Method. We can call it on any objects that belong to the PhoneNumber class.

This post only skims the surface of classes. If you have Chris Pines's "Learn to Program" book, I'd definitely recommend reading Chapter 13. It does a great job of simplifying how classes work. For online resources, I'm always a big fan of The Bastards Book of Ruby. The section on classes is not complete, but still offers up a good first look at how classes work.
