The Basics

For starters, group_by groups an array or hash based on certain criteria. I found the following info on the Global Nerdy blog and thought it explained the basics really well

The Method Expects: A block containing criteria explaining how the collection should be grouped

The Method Returns: A hash in which each key represents an output of the criteria. The corresponding value is an array containing the members of that group

Grouping Arrays

When group_by is used on an array, it iterates through the array and passes each item through the block. The resulting value of the block determines which item is placed into each group. For example, Ruby docs uses the following to illustrate this process:

array = [0..15]group_by {|num| num % 3}
# => 0 => [0, 3, 6, 9, 12, 15]
1 => [1, 4, 7, , 10, 13]
2 => [2, 5, 8, 11, 14]

As you can see, the groups are organized based on the remainder you get when you divide each number by 3.

We can also use different criteria to make our groups. For example, we can take an array and group the items by string length, like this:

bellas = ["Beca", "Aubrey", "Chloe", "Fat Amy", "Cynthia-Rose", "Lilly"]
bellas.group_by { |singer| singer.length}
# => {4=>["Beca"], 6=>["Aubrey"], 5=>["Chloe", "Lilly"], 7=>["Fat Amy"], 12=>["Cynthia-Rose"]}

Grouping Hashes

When group_by is used on a hash, it passes each key-value pair through the block. You can decide whether you'd like your groups to be based on the keys or values. For example:

bellas = {"Beca"=>"alto", "Fat Amy"=>"alto", "Chloe"=>"soprano", "Lilly"=>"alto"}
bellas.group_by { |name, vocal_range| vocal_range}
#=> {"alto"=>[["Beca", "alto"], ["Fat Amy", "alto"], ["Lilly", "alto"]], "soprano"=>[["Chloe", "soprano"]]}
